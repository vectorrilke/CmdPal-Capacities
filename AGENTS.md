# AGENTS

## Scope

This repository is for a Windows Command Palette extension, not a VS Code extension.

## Canonical knowledge base

Use Microsoft Learn Command Palette docs as the source of truth:
- Extensibility model: https://learn.microsoft.com/windows/powertoys/command-palette/extensibility-overview
- Create extension: https://learn.microsoft.com/windows/powertoys/command-palette/creating-an-extension
- Add commands: https://learn.microsoft.com/windows/powertoys/command-palette/adding-commands

General Microsoft technology documentation lookup behavior is defined in:
- .github/instructions/microsoft-docs.instructions.md

## Working rules

- Prefer small, deployable changes.
- For validation, always use Deploy + Reload Command Palette Extension.
- Keep packaging files tracked in source control (launchSettings and pubxml where required by docs).
- Preserve the app manifest COM/AppExtension registration generated by template.

## Typical source layout (after scaffolding)

- src/<ExtensionName>/<ExtensionName>CommandsProvider.cs: top-level commands
- src/<ExtensionName>/Pages/<ExtensionName>Page.cs: list page items
- src/<ExtensionName>/Pages/*.cs: additional commands/pages
- src/<ExtensionName>/Package.appxmanifest: extension registration
- src/<ExtensionName>/Program.cs: startup/host wiring
